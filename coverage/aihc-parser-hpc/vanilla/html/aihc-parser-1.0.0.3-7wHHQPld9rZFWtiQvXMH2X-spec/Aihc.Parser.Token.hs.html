    2 -- Module      : Aihc.Parser.Token
    3 -- Description : Public token API for inspecting lexer output
    4 --
    5 -- This module exposes the stable token-facing surface of the parser without
    6 -- making callers depend on the internal lexer implementation. It is intended
    7 -- for diagnostics, tooling, generators, and tests that need to inspect the
    8 -- token stream directly instead of parsing it into the AST.
    9 --
   10 -- A 'LexToken' records the token 'LexTokenKind', original source text, source
   11 -- span, whether it started at the beginning of a logical line, and its
   12 -- 'TokenOrigin'. Tokens with 'FromSource' come from the input text; tokens with
   13 -- 'InsertedLayout' are virtual braces or semicolons inserted by the layout
   14 -- pass.
   15 --
   16 -- The @lexTokens*@ functions scan arbitrary source fragments. The
   17 -- @lexModuleTokens*@ variants additionally read the module header first, apply
   18 -- any @LANGUAGE@ pragmas (including implied extensions), and then run layout in
   19 -- module mode so top-level declarations receive the same virtual layout tokens
   20 -- as the parser. Use the @WithSourceName@ variants when diagnostics should
   21 -- carry a caller-provided source name instead of @"<input>"@.
   22 --
   23 -- 'readModuleHeaderExtensions' returns the raw extension settings discovered in
   24 -- the leading module pragmas, while 'readModuleHeaderPragmas' separates a
   25 -- language edition from explicit extension toggles.
   26 module Aihc.Parser.Token
   27   ( TokenOrigin (..),
   28     LexToken (..),
   29     LexTokenKind (..),
   30     readModuleHeaderExtensions,
   31     readModuleHeaderPragmas,
   32     lexTokensWithExtensions,
   33     lexModuleTokensWithExtensions,
   34     lexTokensWithSourceNameAndExtensions,
   35     lexModuleTokensWithSourceNameAndExtensions,
   36   )
   37 where
   38 
   39 import Aihc.Parser.Lex
